Инициализация изменений
git init

Добавление изменений в индекс
git add <file or dir name>

Добавление имени изменений
git commit -m "name commit"

Отпрвка изменений на удаленный сервер
git push <name repo> <name branch> (если насильно -f)

Клонирование репозитория
git clone <repo name> (имя копируешь с кнопки код в github)

Стягивание последних изменений с сервера
git pull <name repo> <name branch> (есть опция -rebase, если название веток разное, пригодится на работе)

Отменение коммита, но оставление его в индексе изменений
git reset --soft HEAD~<кол-во коммитов>

Отменение коммита
git reset HEAD~<кол-во коммитов>

Соединять коммиты, переименовывать, менять их местами 
git rebase -i HEAD~<кол-во коммитов>

Обновление ветки с помощью локальной. В branch name вписывается название ветки, которую будут вливать в ту ветку, в которой ты находишься. Если не понятно задай вопрос
git rebase <branch name>

Перенести коммит с другой ветки (пригодится на работе)
git cherry-pick <hash commit>

Спрятать локальные изменения
git stash

Вытащить спрятанные локальные изменения
git stash apply

Удалить спрятанные локальные изменения
git stash drop

Убрать изменения из индекса коммита (когда они в зеленом цвете)
git restore --staged <file name or dir name>

Удалить свои изменения (когда они в красном цвете)
git restore <file name or dir name>


Если что-то упустил буду тут дополнять.
